## 一、署名規範 Attribution

所有論文統一署名：

```
作者 Author: 釋慧鏡 (Shi Huijing)
Project: 指月 Point-to-the-moon
授權 License: CC BY-NC-SA 4.0
```

不使用個人法名、寺院頭銜、宗派傳承。法不屬於任何人，研究以內容本身說話。

> 正法眼藏，不立文字——法的價值在法本身，不在說法者的身份。

## 二、引用原則 Reference Policy

### 2.1 佛教典籍：僅用大正藏原典

**必須引用：** 大正藏（Taishō Tripiṭaka）原典，標明冊號與編號。

```
格式：《經論名》，造/譯者，《大正藏》第X冊，No. XXXX。
範例：《成唯識論》，護法等菩薩造，唐玄奘譯，《大正藏》第31冊，No. 1585。
```

所有經論引文須經 CBETA 電子佛典集成校對，並附 CBETA wiki-link（如有）。

**不引用：** 近現代中文佛教學者的詮釋性著作。原因：

- 近現代漢傳佛教學術圈存在宗派立場爭議，引用任何一家都可能引發非學術性的爭論
- 本研究直接以原典為依據，讀者可自行驗證，不需要經過任何中間詮釋者
- 這也是「諸法實相」開源精神的體現——回到源頭，如實呈現

**例外：** 古代祖師的注疏（如窺基《成唯識論述記》、澄觀《華嚴經疏》）屬於經典傳統的一部分，可以引用，視同原典層級。

### 2.2 現代學術著作：英文為主

現代學科（心理學、神經科學、物理學、哲學、複雜系統理論等）的引用，以英文同行評審學術著作為主。

```
格式：Author, A. B. "Title." Journal/Book, Publisher, Year.
範例：Vaswani, A. et al. "Attention Is All You Need." Advances in Neural Information Processing Systems 30, 2017.
```

原因：

- 心理學（CBT/ACT）、神經科學、AI 研究等領域的原始文獻幾乎全部以英文發表
- 英文學術著作有明確的同行評審機制，品質可驗證
- 面向國際讀者，英文引用直接可查

### 2.3 引用的三層分類

參考文獻統一分為三類：

```markdown
## 參考文獻

### 一、佛教原典 Primary Buddhist Sources
（大正藏原典 + 古代祖師注疏）

### 二、現代學術著作 Modern Scholarship
（英文同行評審學術著作）

### 三、經論索引 Sūtra & Śāstra Index
（表格形式，列出本文引用的所有經論簡稱、全名、大正藏編號、造/譯者）
```

## 三、論文結構 Paper Structure

所有論文遵循以下統一結構：

### 3.1 標題區 Header Block

```markdown
# 中文主標題

**中文副標題**

*English Main Title:
English Subtitle*

---

**作者 Author:** 釋慧鏡 (Shi Huijing)
**日期 Date:** 2026年X月
**授權 License:** CC BY-NC-SA 4.0
**版本 Version:** 1.0

---
```

### 3.2 摘要 Abstract

雙語呈現，先中後英。摘要應包含：核心命題、論證路線、主要結論。

**三句話結構**（每句可展開為 2-3 行，總計中文 200-300 字，英文 150-250 words）：

1. **問題**：本文要回答什麼問題，為什麼這個問題重要
2. **論證**：本文如何論證（用什麼經典依據、什麼分析框架）
3. **結論**：核心發現是什麼，對修行或理解有什麼意義

```markdown
## 摘要 Abstract

（中文摘要，200-300字，三句話結構）

（English abstract, 150-250 words）

**關鍵詞 Keywords:** 中文詞1、中文詞2 / English term 1, English term 2
```

### 3.3 正文章節 Body Sections

主節用中文數字（一、二、三），子節用阿拉伯數字（2.1, 2.2, 2.3）。

```markdown
## 一、問題意識

正文……

## 二、核心論證

### 2.1 第一層論證

正文……

### 2.2 第二層論證

正文……
```

理由：主節的中文數字保持學術中文的傳統感，子節的阿拉伯數字便於跨語言引用和精確指涉。

### 3.4 必備章節 Required Sections

每篇論文必須包含以下章節（順序固定）：

| 順序 | 章節 | 說明 |
|------|------|------|
| 1 | 問題意識 / 引言 | 為什麼這個問題重要，本文要回答什麼。**第一段必須有鉤子**（見下方說明） |
| 2 | 經典依據 | 大正藏原典引文，建立義理基礎 |
| 3 | 核心論證 | 本文的原創分析，可分多節 |
| 4 | 跨學科會通 | 現代學科的結構性類比。須有深度（見下方說明），否則寧可不寫 |
| 5 | 中道校正 | 防止過度詮釋，標明命題的邊界與限制 |
| 6 | 結論 | 核心論點精要 + **修行指引** + **開放問題**（見下方說明） |
| 7 | 參考文獻 | 三層分類（見上） |
| 8 | 經論索引 | 表格形式，列出本文引用的所有經論 |
| 9 | 系列論文交叉引用索引 | S1-P04 起，標明與其他論文的義理關聯 |
| 10 | 腳注 | 集中在文末，以「## 腳注」為標題 |
| 11 | Footer | CC BY-NC-SA 4.0 + CBETA 校對 + GitHub（3 行） |

**中道校正**是本研究的方法論特色，每篇必須有。至少包含：
- 防止義理面的過度延伸
- 防止跨學科類比的還原主義
- 標明「啟發性類比」(heuristic) vs「理論等價」(reductive) 的邊界

### 3.4.1 問題意識的鉤子 Opening Hook

問題意識的**第一段**必須有一個鉤子（hook），讓讀者立刻產生共鳴。目標讀者是 65+ 的佛教修行人，不是學術圈——先有共鳴，再談義理。

**有效的鉤子類型：**
- **日常場景**：「你有沒有這樣的經驗——明明知道不該生氣，卻控制不住？」
- **反直覺問題**：「看得懂就是看得懂，看不懂就是看不懂——這句廢話，其實是金剛經的教學設計。」
- **修行困惑**：「打坐三十年，為什麼還是煩惱不斷？」

**避免的開頭：** 直接從定義或學術背景開始（「百法明門論是世親菩薩所造...」）。

### 3.4.2 跨學科會通的深度要求 Interdisciplinary Depth

跨學科會通不是「點到為止的類比」。如果要寫，必須有三層結構：

1. **相似處**：佛法的 X 概念與現代的 Y 理論在什麼維度上相似
2. **根本差異**：但它們在什麼層面上不同（通常是：目的不同、本體論不同、修行 vs 治療）
3. **不可還原聲明**：明確說明這是「啟發性類比」(heuristic)，不是「理論等價」(reductive)

**字數門檻：** 至少 500 字。如果找不到有深度的類比，**直接砍掉整個章節**——寧可不寫，也不要湊數。淺薄的類比比沒有類比更有害，因為它暗示佛法可以被現代理論還原。

### 3.4.3 結論的三層結構 Conclusion Structure

結論不是摘要的重複。必須包含三層：

1. **精要重述**（2-3 句）：本文的核心發現，用最精煉的語言重新表達
2. **修行指引**（so what for practice）：這個義理對日常修行有什麼具體啟發
3. **開放問題**：留一個問題給下一篇論文或讀者自己思考——讓讀者帶著問題離開

### 3.5 腳注 Footnotes

使用 `[^n]` 格式。腳注用於：
- 經文引用的出處標註
- 學術異說的標注（如種子論的護法 vs 難陀 vs 勝軍三系異說）
- 跨體系引用的說明（如同時引用如來藏系和唯識系時的理論框架差異）
- 術語的精確定義

不用腳注做一般性的補充說明——那些內容應該寫進正文。

**位置規則：** 腳注定義全部集中在文末，以 `## 腳注` 為標題，位於「系列論文交叉引用索引」之後、Footer 之前。**不得散落在各章節內。**

**格式規則（引文為主，造譯者不放腳注）：**

```markdown
[^1]: 《經論名》卷X，「引文原文片段」，引文脈絡說明，《大正藏》第X冊，No. XXXX。
```

範例：

```markdown
[^1]: 《般若波羅蜜多心經幽贊》卷上，「心者，堅實妙最之稱」，《大正藏》第33冊，No. 1710。
[^2]: 《成唯識論》卷九，「轉捨依他起上遍計所執，及能轉得依他起中圓成實性」，轉依義，《大正藏》第31冊，No. 1585。
```

造者/譯者資訊統一放在「參考文獻」區，不在腳注中重複。腳注專注於**引文片段 + 脈絡說明 + 出處定位**。

### 3.6 經文引用格式 Sūtra Quotation Format

經文引用使用 blockquote，引用後標明出處腳注：

```markdown
> 「初發心時，便成正覺，知一切法，真實之相。」[^1]

[^1]: 《大方廣佛華嚴經》卷十七，初發心功德品第十七，《大正藏》第10冊，No. 278。
```

梵文術語首次出現時附梵文原文，格式：中文（梵文），如：作意（manaskāra）。

## 三之二、寫作方法論 Writing Methodology

### 3.7 RAG 先行原則 RAG-First Principle

每篇論文開寫前，**必須先用 sub-agent 平行 vault 搜索進行系統性查詢**。

> **重要：不使用 AnythingLLM API**。實測 AnythingLLM RAG 有 ~22% 幻覺率，而 sub-agent 直接 grep vault markdown 檔案的幻覺率為零。（Lesson #3, 2026-03-23）

**流程：**

1. **提取核心概念**：從論文大綱提取所有需要經典支撐的義理命題
2. **設計查詢**：每個命題設計 2-3 條查詢，涵蓋不同角度（見 §3.7.2 查詢模板）
3. **平行搜索**：啟動 sub-agent（每 agent 負責一個 vault 目錄），用 grep 搜索 `~/Projects/tripitaka/vaults/` 下的 markdown 檔案
4. **篩選命中**：人工審核結果，確認出處準確、語境相關，用標準格式記錄（見 §3.7.3）
5. **CBETA 交叉驗證**：所有 RAG 結果必須與 CBETA 原典交叉驗證後才能引用
6. **依據寫作**：論文正文的每一條經文引用，都必須可追溯到 RAG 結果或 CBETA 原典

RAG 結果檔案保留在本地（`series{N}/rag/paper{N}_rag_{group}.md`），不推送到 GitHub。

**搜索最佳實踐：**
- 指定 T-number + 關鍵術語（命中率 85%+）
- 指定 vault 路徑（如 `vault-般若/`、`vault-唯識/`）
- 核心概念強制至少 3 角度查詢（定義、因果、修行）
- 非核心概念可只查 1 角度

### 3.7.1 Vault 缺口預審 Vault Coverage Audit

**每個系列開寫前**，必須做一次 Vault Coverage Audit，避免寫到一半發現核心經論不在 vault 裡。

**流程：**

1. 列出本系列預計引用的核心經論清單（10-20 部），含 T-number
2. 跑 `ls ~/Projects/tripitaka/vaults/vault-*/` 確認哪些在、哪些不在
3. 對每個缺口做出決定並記錄：
   - **(a) 補進 vault**：從 CBETA 下載轉換後加入對應 vault
   - **(b) Web CBETA 替代**：用 CBETA Online 手動查閱，不依賴本地 vault
   - **(c) 放棄引用**：調整論文架構，不引用該經論
4. 產出：`series{N}/vault_audit.md`

```markdown
# Series {N} Vault Coverage Audit

| 經論 | T-number | Vault 路徑 | 狀態 | 備註 |
|------|----------|-----------|------|------|
| 成唯識論 | T1585 | vault-唯識/ | ✅ 完整 | |
| 大般若經 | T0220 | — | ❌ 缺失 | 600 卷，用 web CBETA 替代 |
```

### 3.7.2 RAG 查詢模板 Query Templates

四類標準查詢模板，依需要組合使用（詳見 `research/rag_templates.md`）：

| 類型 | 模板 | 使用時機 |
|------|------|---------|
| 定義類 | 「{經論名}（{T-number}）中關於{術語}的定義段落」 | 需要精確定義時 |
| 修行次第 | 「{經論名}中{修行法門}的次第與階位」 | 涉及修行步驟時 |
| 對治法門 | 「{經論名}中以{法門}對治{煩惱}的段落」 | 討論實踐應用時 |
| 宗義比較 | 「{術語}在{經論A}和{經論B}中的用法差異」 | 跨文本分析時 |

### 3.7.3 RAG 結果標準格式 Result Format

所有 RAG 結果統一使用以下格式記錄：

```markdown
## Q{N}: {查詢描述}
- **查詢**: {完整查詢文字}
- **Vault**: {搜尋範圍，如 vault-唯識/}
- **命中**: ✅ 優 / ⚠️ 良 / ❌ 差 / 🔲 未命中
- **來源**: {T-number, 卷次, 行號或檔名}
- **引文**: > 「原文片段」
- **備註**: {為什麼選用/不選用，是否需要 CBETA 二次驗證}
```

**品質分級標準：**
- **✅ 優**：完整段落，出處明確，可直接引用
- **⚠️ 良**：部分匹配，語境清楚，需要補充上下文
- **❌ 差**：無關結果、錯誤文本、幻覺
- **🔲 未命中**：目標文本不在 vault 中

### 3.8 雙語同步 Bilingual Dual-Track

每篇論文同時產出中文版與英文版：

- **中文版**為主體，英文版為學術翻譯（非逐字直譯）
- 兩版共享相同的章節結構、腳注編號、經論索引
- 英文版的經文引用保留中文原文，附英文翻譯
- 兩版的結尾聲明（CC BY-NC-SA、CBETA 驗證、GitHub repo）必須同步

```
papers/
├── 覺察即轉向.md                    ← 中文版
├── awareness-as-turning-point.md    ← 英文版
```

### 3.9 系列論文交叉引用 Cross-Reference Index

從 S1-P04 起，每篇論文末尾附「系列論文交叉引用索引」，標明與其他論文的義理關聯：

```markdown
## 系列論文交叉引用索引

| 論文 | 與本文關係 |
|------|-----------|
| S1-P03 大型語言模型的唯識定位 | 一破一立：S1-P03 判定 AI 不具作意，S1-P06 論證人如何運用作意覺醒 |
```

**一破一立（破-立 pairing）**是本系列的核心結構模式：
- S1-P03（破）→ S1-P06（立）：作意判準
- S1-P04（破）→ S1-P05（立）：增上緣善用

每篇新論文應明確標示自己在系列中的位置與關聯。

### 3.10 經論索引表 Sūtra & Śāstra Index Table

每篇論文末尾必須附經論索引表，格式統一：

```markdown
| 序號 | 經論名 | 大正藏冊號 | 編號 | 本文引用卷次 |
|------|--------|-----------|------|-------------|
| 1 | 成唯識論 | T31 | No. 1585 | 卷二、三、四 |
```

索引表必須與腳注實際引用完全吻合——推送前進行交叉審計。

### 3.11 白話生活譬喻 Vernacular Analogy Pattern

每篇論文在核心論證段落應包含至少一個**生活化的白話譬喻**，將抽象義理轉化為讀者可直接體會的場景。

S1-P06 範例：「狗對你吠」譬喻——

- 非理作意 = 被狗吠嚇到，心跳加速，憤怒回罵（業力自動駕駛）
- 如理作意 = 聽到狗吠，覺察「這是聲音、這是我的反應」，選擇不被帶走

原則：
- 譬喻必須是日常場景（不是另一個佛學概念）
- 先呈現「未覺察」的反應模式，再呈現「覺察後」的轉變
- 譬喻服務義理，不喧賓奪主
- **倫理審查**：所有譬喻發布前必須檢查是否違反佛教戒律。例：涉及葷食的譬喻一律替換為素食（滷肉飯→陽春麵）。（Lesson #6）

### 3.12 寫作語氣規範 Tone & Voice Rules

**一核心一論文（Lesson #7）：** 每篇論文只論證一個核心命題。如果寫著寫著發現範圍擴大（同時在講法器、八正道、正念），立刻停下來拆分。聚焦後的論文邏輯永遠比面面俱到的論文清楚。

**中文版禁用隨意英文（Lesson #4）：** 中文版正文不混入英文。學術術語可用括號標註（如：作意（manaskāra）），但不可在行文中隨意穿插英文詞彙。目標讀者年齡 65+，混合語言嚴重影響閱讀體驗。

**尊重讀者，不貼標籤（Lesson #5）：** 不直接對讀者下判斷（如「你有漏」「你是凡夫」）。用問題引導讀者自我發現，用譬喻讓讀者自己對號入座。佛法是鏡子，不是標籤機。

**跨系列語氣切換：** S3（唯識）偏精確分析，S5（般若）偏敘事流動。跨系列論文（如 S5-P10 橋接）應自覺 toggle——引唯識時用工具箱語氣，引般若時用流動隱喻，不硬統一。

## 四、分析框架標記 Analytical Conventions

沿用百法明門論研究建立的標記體系：

| 標記 | 意義 | 使用場景 |
|------|------|---------|
| ✅ 義理可立 | 經論依據充分，邏輯自洽 | 核心命題判定 |
| ⚠️ 假說比喻 | 啟發性類比，非理論等價 | 跨學科會通 |
| ❌ 邏輯有誤 | 常見誤解，需要校正 | 中道校正 |

## 五、文件命名與目錄結構 File Naming & Directory Structure

### 5.1 發布目錄（推送到 GitHub）
